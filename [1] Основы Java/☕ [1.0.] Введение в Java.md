# ☕ [1.0] Введение в Java
___
## 🔼 [1.0.1] Введение
### Представим ситуацию.
### Мы хотим приготовить пиццу.
### Нельзя просто сказать: "Приготовить пиццу".
### Нам нужны точные инструкции:
### 1. Взять тесто
### 2. Раскатать его
### 3. Добавить соус.
### и т.д.
### То есть, инструкция - это последовательность команд.
### `Программа` - такая же инструкция, только для компьютера.
### `Язык программирования` - это язык, на котором человек записывает эту инструкцию.
###
___
## ☕ [1.0.2] Java
### `Java` - это объектно-ориентированный язык программирований общего назначения, созданный с упором на переносимость, надежность и безопасность.
### Java `платформонезависима`, поскольку программа, написинная на Java может запускаться на любоф платформе, будь-то Windows, Linux, macOS и другие, благодаря байткоду и JVM. При условии, что для данной операционной системы есть версия JVM.
___
## ❓ [1.0.3] Почему нельзя писать сразу на языке компьютера
### Компьютер не понимает слова вроде: `System.out.print("Привет");`
### Для него существует только два значения:
### `0` 
### `1`
### Это называется машинный код
### Например, одна команда процессора может выглядеть так:
### `10110000 01100001`
### Человеку такое читать практически невозможно.
### Поэтому, придумали языки высокого уровня.
### И один из них Java.
___
## ❓ [1.0.4] Почему Java может работать почти везде
### Большинство языков программирования делают так:
### `Код ➡︎ Сразу машинный код ➡︎ Программа`
### То есть программа компилируется отдельно под Windows, отдельно под Linux, отдельно под macOS.
### Java работает иначе.
### Она использует промежуточный язык.
### Схема выглядит так:
    Main.java (Java-код)
            ⬇︎
    javac (Компилятор)
            ⬇︎
    Main.class (Байткод)
            ⬇︎
            JVM
            ⬇︎
    Машинный код
### Именно это позволяет запускать одну и ту же программу на разных ОС.
___
## 💬 [1.0.5] Компилятор
### `Компилятор` - это программа, которая переводит код, написанный человеком, в форму, пригодную для выполнения компьютером.
### В Java эту задачу выполняет компилятор `javac`
### Например, есть файл `Main.java`
### Когда запускается компилятор: `javac Main.java`
### Он не создает машинный код.
### Он создает файл: `Main.class`
### Этот файл содержит байткод.
___
## 📄 [1.0.6] Байткод
### `Байткод` - это промежуточный код, который не зависит от операционной системы и компьютера.
### Это не Java-код, не машинный код, это промежуточное представление программы.
### Именно поэтому один и тот же файл `.class` можно запускать на разных платформах, если на них установлена JVM.
___
## ⚙️ [1.0.7] JVM
### `JVM` - это виртуальная машина Java.
### Это обычная программа, которая запускается на компьютере и выполняет байткод.
### Она является самым важным элементом всей платформы Java
### Без JVM файл `.class` запустить нельзя.
### Он выполняет следующее:
### 1) Загружает классы (Class Loader)
### 2) Проверяет байткод
### 3) Выделяет память
### 4) Запускает Garbage Collector
### и т.д.
___
## 🖥️ [1.0.8] JRE
### `JRE` - это среда выполнения Java.
### Если сказать проще:
### `JRE - это все, что нужно, чтобы запустить Java-программы.`
### В состав JRE входят: JVM, стандартные библиотеки и файлы необходимые для выполнения программы.
___
## 🛠️ [1.0.9] JDK
### `JDK` - это набор инструментов для разработки Java-программы.
### Он нужен для разработки программы.
### В состав JDK входят: JRE, javac и инструменты разработки
___

## 📋 [1.0.11] Схема выполнения программы
### Соберем все в единое.
### Представим, мы написали программу:
    public class Main {
        public static void main(String[] args) {
             System.out.println("Привет!");
        }
    }
### 1) Запускаем файл `Main.java`. Это обычный текстовый файл с исходным кодом.
### 2) Компилятор `javac` читает этот файл. Если ошибок нет, он создает: `Main.class`. Внутри находится байткод.
### 3) JVM загружает файл `Main.class`
### 4) JVM проверяет байткод.
### 5) JVM преобразует байткод в машинные инструкции для процессора
### 6) Процессор выполняет эти инструкции

# `☕ [1.0] Введение в Java`
#### Представим, что мы хотим приготовить пиццу. Мы не можем просто сказать повару: "Приготовь пиццу!". Ему нужны точные инструкции:
#### 1. Взять тесто,
#### 2. Раскатать его
#### 3. Добавить соус
#### и т.д.
#### Программа такая же инструкция, только для компьютера.
### `🗯️ | Язык программирования - это язык, на котором человек записывает эту ниструкцию.`
#### Но компьютер не понимает язык Java. Для него существует только два значения: 0 и 1.
#### Это называется `машинный код.`
### `💬 | Компилятор - это программа, которая переносит код, написанный человеком, в форму, пригодную для выполнения компьютера.`
#### В Java эту задачу выполняет программа `javac`
#### Например, у нас есть файл `Main.java`. Когда запускается компилятор `javac`, он не создаёт машинный код. Он создает файл: `Main.class`. Этот файл содержит в себе байткод.
### `📄 | Байткод - это промежуточный код между Java и машинным, который не зависит от операционной системы и компьютера.`
#### Важно понимать, что это не Java-код, не машинный код. Это промежуточное представление программы.
### `⚙️ | JVM - это виртуальная машина Java.`
#### Это обычная программа, которая запускается на компьютере и выполняет байткод.
#### Но она не просто запускает программу. Ее основные задачи:
#### 1. Загружает классы
#### 2. Проверяет байткод
#### 3. Управляет памятью
#### 4. Выполняет байткод
#### и т.д.
### `▶️ | JRE - это среда выполнения Java.`
#### Если сказать проще, то это все что нужно для запуска Java-программы.
#### В состав JRE входят JVM, стандартные библиотеки Java и файлы необходимые для выполнения программы.
### `🛠️ | JDK - это набор инструментов для разработки Java-программы.`
#### В состав JDK входят JRE, компилятор javac и инструменты разработки.
#### Он нужен тем, кто пишет программы.
#### Общую структуру можно представить так:
    JDK
    |_javac
    |_javadoc
    |_jar
    |_jdb
    |_JRE
        |_JVM
        |_Библиотеки Java
